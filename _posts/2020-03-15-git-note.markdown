之前在实习的时候，公司项目代码用Gitlab托管。经过一段时间的摸索，用git对代码进行维护可以总结成下面几步：

- git status 查看相关文件的状态

- git add filename 将后方紧跟的文件进行暂存，以便commit使用。

- git commit -m "[tb] add directory_name" 对暂存区的文件进行提交到本地仓库，并添加注释信息。

- git pull --rebase 拉取远程仓库的文件，可能其他人也对代码进行更新

- git push 将修改后的文件推送到远程仓库

出现conflict之后，修改file内容，git add之后直接git rebase --continue
